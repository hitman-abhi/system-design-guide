# Intermediate

This section is for topics that assume the reader already understands the core system design foundations.

These topics usually focus on:

- composing multiple building blocks together
- operating distributed systems under realistic failure modes
- reasoning about tradeoffs in system behavior, not just definitions

- quorum
- leader election
- distributed locks
- service discovery
- circuit breakers
- backpressure
- load shedding
- batching
- pub/sub
- event-driven architecture
- saga pattern
- outbox pattern
- CQRS
- read replicas
- write-ahead logs
- schema evolution
- secondary indexes in distributed systems
- pagination strategies
- API gateways
- webhooks
- CDN design
- failover strategies
- disaster recovery basics
- multi-region design
- security basics for distributed systems
- authentication vs authorization
- session management
- connection pooling
